Difference between id and class¬±

    üîπ id:

  • Used to uniquely identify a single HTML element.
  • Must be unique within a page (only one element can have a specific id).
  • Selected in CSS using #idName.
  • Commonly used for JavaScript targeting: document.getElementById("idName").
  • Has higher specificity in CSS.
  • Generally used for individual, one-off elements (e.g., a header, a specific button).

    • üîπ class:

  • Used to define a group of elements with common styling or behavior.
  • Can be reused across multiple elements.
  • Selected in CSS using .className.
  • Accessed in JavaScript with document.getElementsByClassName() or querySelectorAll().
  • Has lower specificity than id in CSS.
  • Suitable for applying the same style or behavior to multiple elements.

What is a <summary> elememt?üìö

The <summary> element in HTML is used as a summary, heading, or label for the <details> element. It acts like a clickable toggle that shows or hides additional content.

    üîπ Key Points:

  1. <summary> must be the first child of a <details> element.
  2. When clicked, it toggles the visibility of the content inside the <details>.
  3. Useful for collapsible sections like FAQs, disclosures, or expandable info.

What is CSS?üé®

  • CSS stands for Cascading Style Sheets.
  • It is a language used to describe the style and layout of HTML documents.

    üîπ Key Points about CSS:

  1. Controls the look and feel of a webpage (e.g., colors, fonts, spacing, positioning).
  2. Keeps content (HTML) and presentation (CSS) separate for better organization and reusability.
